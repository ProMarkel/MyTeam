## 1. MVP 版本定义

### 1.1 核心目标
- **跑通两个循环（建生产线）**：① 锁定 1 个核心循环（合成→经营→解锁剧情钩子）与美术基调；完成封闭新手教学 + 开放操作 1–2 关 / 基础循环（≈5 分钟达成"有效用户"）。② 搭建 AI 短剧内容引擎生产线（脚本模板、分镜流水线、精修 SOP）——按"可复用生产线"标准搭建。
- **阶段纪律**：MVP 阶段不设对外闸门，重点"把两条生产线建起来"；内部以 **"可玩、逻辑自洽、首片过审"** 为完成判据。

### 1.2 关键交付
1. **可玩核心循环原型**（产品侧，另由程序 / 策划交付）
2. **第 1 条 AI 短剧成片过审**（须过精修红线，见 §3.6）
3. **内容引擎 SOP 文档**（即「AI 短剧制作 SOP」，见 §3）

### 1.3 MVP 范围
**做这些（MVP 硬边界）**
- **1 个核心循环**：合成（食材 / 器物发射 → 菜品 / 道具）→ 经营（交付订单→获得资源→修葺店铺→扩张），闭环清晰（详见 §2；目标态 4 类 20 链，MVP 切片 5 条）
- **古风餐厅皮**：单场景、基础美术符号体系；美术基调为融合唐宋明的架空古风时代、Q 版 2.5~3 头身 + 豆豆眼；食魂 mascot 作为合成引导角色
- **合成链切片（MVP）+ 5 关**：MVP 建代表性切片验证机制（食材×2+器物×1+菜品×1+道具×1，各开放前 3–4 层，详见 §2.2）；目标态 20 链（8 食材+4 器物+4 菜品+4 道具）
- **1 条主线剧情钩子**
- **激励视频 IAA 接入**（首日第一变现途径，验证广告变现基线）+ **基础 IAP**（去广告 / 体力包）
- **短剧→游转化埋点**：分镜即埋钩、剧末 CTA、来源追踪

**砍掉清单（不做这些，留待阶段③ / ④ 验证后再加）**
社交 / 公会 ｜ 多端同步 ｜ UGC ｜ 复杂装修系统 ｜ 多角色线 ｜ 排行榜 ｜ 复杂分支剧情 ｜ 大量美术变体 ｜ 重度 IAP 系统 ｜ 长线运营活动。

**变现前提**：IAA 占总 LTV ≤ 5%；不破坏首日体验。

> **路线图位置与周期**：团队就位（起点）→ **1 MVP 版本（建生产线）** → 2 3 日版本 → 3 7 日版本 → 4 长线盯数 → 5 拓展产品（箭头过节点 5 向右延伸）。团队就位 ~ MVP 版本 **6~8 周**；MVP 阶段周期 6 人团队 T0–T0+8 周 / 9 人团队 T0–T0+6 周。详细路线、阶段总表与节点细化见「项目方案概览」§6.1 / §6.2 / §6.3（即 `项目计划书.md` §6）。

### 1.4 变现嵌入点（整合自「核心循环设计卡」§9）
- **IAA**：激励视频（复活 / 体力 / 加速），插屏节制使用保体验
- **IAP**：去广告包 ¥6–12；体力 / 资源包 ¥1–30；限时古风皮肤 / 餐厅主题 ¥6–18
- 嵌入时机前提见 §1.3（IAA 占总 LTV ≤ 5%，不破坏首日体验）
- **MVP 阶段说明**：上述付费点位在 MVP 中主要为**体验感受与版号版本申报需要**而预留界面与交互路径，**不真实接入 IAA / IAP 后端结算功能**。实际商业化接入留待阶段③/④验证后再接通。

## 2. 核心循环原型

### 2.1 核心玩法循环
```
合成（食材/器物） → 经营（交付订单/修葺店铺） → 解锁剧情钩子 → 追剧/分享裂变 → 次日回流
```
餐厅具体实例化：合成食材 → 交付订单换取资源 → 修葺店铺推进经营任务 → 推动剧情 → 解锁更多合成元素。
- **待细化**：每一环的"用户动作 + 正反馈 + 单环时长"，串成一条 5 分钟不冷场的体验链。

### 2.2 合成链设计

**基础规则（二合连续单线）**
- 每级消耗 **2 个上一级产物** → 产出 1 个下一级；每条链 `A→B→C→…` 单一线性、无分叉。
- 层数 n 的链，产 1 个顶层需 `2^(n-1)` 个最底层材料——**复利成本是节奏旋钮**：链越长，单链时长越长。

**四类合成链（目标态架构）**

| 类别 | 数量 | 单链长度（目标态） | 性质 | 发射器行为 |
| --- | --- | --- | --- | --- |
| 食材链 | ×8（主食 / 蔬菜 / 禽类 / 肉类 / 鱼类 / 虾蟹 / 水果 / 香料，分 2 组各 4） | 8–12 | 原料合成线 | 本身不发射，作为被发射物 |
| 器物链 | ×4（菜篮 / 渔具 / 畜栏 / 货架） | 10（"以此类推"可延伸） | 永久发射器（有 CD，不消失） | 见「器物节点发射阶梯」 |
| 菜品链 | ×4 | 6–8 | 消耗性发射器（顶阶食材化身，发射 N 次后消失） | 发射菜品 |
| 道具链 | ×4（宝箱 / 礼盒 / 体力瓶等） | 2–4 | 资源发射器 | 发射金币 / 钻石 / 体力 / 任意链物品；长度影响列表与几率 |

**器物节点发射阶梯（单链长度 10 为例）**
- 节点 1–3：不发射食材
- 节点 4：发射 食材链组1 · 物品1
- 节点 5：+ 连续发射数量
- 节点 6：+ 食材链组2 · 物品1
- 节点 7：+ 几率发射 食材链组1 · 物品2
- 节点 8：+ 几率发射 食材链组2 · 物品2
- 节点 9：+ 连续发射数量
- 节点 10：提升 两组食材链物品2 发射几率
- （"以此类推"：更高节点延续 **扩组 → 扩物品 → 提量 → 提几率** 四拍循环）

**MVP 切片（硬边界，对齐 §1.3 / 5 分钟闭环）**
- MVP **不铺满 20 链**，仅验证"二合 + 发射器"机制闭环，建议切片：
  **食材链 ×2 + 器物链 ×1 + 菜品链 ×1 + 道具链 ×1 = 5 条代表性链**，每条**仅开放前 3–4 层**（顶阶不进 MVP）。
- 用这 5 条证明链路自洽：合成 → 器物发射食材 → 菜品消耗性发射 → 道具补给 → 经营 → 解锁回流；通过后再按目标态扩到 8 / 4 / 4 / 4 与全长。

**平衡约束**
- 二合复利下，单链长度 8–12 → 顶层需 `2^7 ~ 2^11 = 128 ~ 2048` 个底层材料；MVP 仅暴露前 3 层（≤4 个底层），把"长度"当长期目标，保障 5 分钟首循环可达成。

- **钩子**：合成产物即剧情道具；「食魂」作为合成 mascot NPC 串联短剧与游戏。

### 2.3 经营循环
- **交付订单（食材 or 菜品）⇒ 获得资源**：将合成产出的食材 / 菜品交付顾客订单，换取金币 / 钻石 / 体力等资源。
- **修葺店铺 ⇒ 完成经营任务**：用资源推进店铺修葺（非"装修"泛概念），每阶段修葺目标对应一项经营任务；完成修葺即推进剧情与解锁。
- 经营提供的"目标感与长线"与合成正反馈在"交付→修葺→解锁"上闭环，避免两段体验割裂。

### 2.4 解锁剧情
- 1 条主线剧情钩子，卡点弹剧情
- 短剧→游 CTR **≥ 3%**（硬指标，低于即触发验证体系黄灯）
- 钩子具体埋点：分镜埋钩 / 剧末 CTA / 角色客串（见 §3.7 成片发布·埋点）

### 2.5 新手 5 分钟路径
```
封闭教学（开场剧情导入 → 场景展示 → 进入合成棋牌 → 器物合成×1 → 食材发射×1 → 食材合成×2 → 交付订单×2 → 修葺店铺×1）
  → 开放操作（解锁剧情 → 继续循环×2）
  → 体验时长 ≈ 5 分钟 = 有效用户
```
- **封闭教学边界**：从「开场剧情导入」到第一个「修葺店铺 ×1」完成为封闭教学（强制引导）；其后（解锁剧情 / 继续循环）转为开放操作。
- **衔接 AI 短剧**：开场剧情导入承接 AI 短剧末尾 CTA，做到剧游无缝衔接。
- **待细化**：分步教学脚本、卡点节奏、首屏 30 秒留存设计。

## 3. AI 短剧制作 SOP（8 步生产线 · 对齐 AI短剧制作SOP.md）

> 来源与权威详版：「AI 短剧制作 SOP」v3.0（根目录 `AI短剧制作SOP.md`）。本节约为其 MVP 设计口径的镜像摘要——完整钩子库、大女主创作圣经、分镜模板、精修红线、三色灯阈值以该 SOP 为准。引擎按"**换皮出海零成本复用**"标准搭建，换题材只换皮、不换骨架。关键纪律：每条短剧在**分镜阶段就埋好游戏钩子**，不是成片后硬贴链接。

### 3.1 剧本选题（主题库 + 钩子库）
- **工具**：飞书多维表格 / Notion（主题×钩子双表）+ 对话式大模型（GPT-5 / Claude / Kimi）扩写钩子。
- **双层钩子库**：通用钩子（打脸爽点 / 逆袭高光 / 公平反击 / 独立清醒 / 悬念卡点，跨主题复用）+ 特定钩子（按五大母题绑定游戏节点）。
- **五大母题（大女主向）**：① 厨艺打脸 ② 身世逆袭 ③ 经营危机 ④ 合成彩蛋 ⑤ 高光节日；每条映射一个游戏内解锁节点（剧情钩子→次日回流）。
- **主题储备纪律**：主投「大女主古风餐厅逆袭」；常备 ≥2 套备用主题（含换皮出海「中世纪女厨酒馆逆袭」），红灯触发 24h 内切换。

### 3.2 剧情脚本（分镜细化）
- **工具**：GPT-5（备选 Claude / Kimi）+ 分镜模板。
- 每条 ≤60s，前 3 秒有钩子、剧末 CTA 清晰；结构铁律：钩子前置（镜 1–3）、情绪中段、悬念卡点收尾（剧末=游戏入口）。
- **大女主红线**：苏望月为唯一核心驱动者，女主须为行动发起者，禁"等男主救 / 恋爱脑"；游戏钩子栏分镜阶段必填。

### 3.3 角色 & 场景设定
- **工具**：GPT-Image-2（角色卡 + 场景概念图，固定 seed + 参考图保一致）。
- **大女主阵容（游戏内复用）**：苏望月（大女主·食坊主理 / 玩家化身）、赵氏（继母反派）、陆三郎（退婚渣男）、金满楼掌柜（同行打压）、小满（学徒厨娘）、老顾（退休名厨·合成链解锁）、食魂（合成 mascot）。

### 3.4 静态分镜生成
- **工具**：GPT-Image-2（关键帧，复用 §3.3 视觉档案保同镜人物一致）。
- 逐镜生成，非卡通廉价感（唐宋明架空基调）。

### 3.5 动态视频生成
- **工具**：SeeDance 2（图生视频 / 文生视频，口型同步，动作自然不抽搐）→ 粗剪视频。

### 3.6 剪辑配音后期（精修红线）
- **工具**：剪映 / CapCut（AI 配音 + 智能字幕 + 调色）。
- **精修红线（任一不达标打回重做）**：画面（无脸崩 / 手指异常 / 廉价感）、节奏（前 3 秒钩子、≤60s、每 8–10s 情绪点）、声音（配音清晰、BGM 契合古风）、转化（剧末游戏钩子 + CTA 清晰）。

### 3.7 成片发布（埋点）
- **平台**：抖音 / 视频号 / 小红书 / TikTok Minis（出海后）。
- **游戏转化埋点（三处一致）**：评论区置顶 + 视频简介 + 剧内口播；分镜即埋钩（望月手撕反派时说"这店我开定了，配方在我手里"自然引导进游戏）、剧末卡点停在游戏可继续处、短剧角色游戏内 NPC 闭环。
- **硬指标**：短剧→游 CTR ≥ 3%（低于触发 §3.8 黄灯）。周更 2–3 条（一/三/五发，周末复盘）。

### 3.8 数据反馈（三色灯）
- **工具**：平台后台 + 飞书多维表格看板（CTR / 完播率 / 涨粉·自然量占比 / 进游注册数）。
- **三色灯**：🟢 CTR≥3% 且完播≥基线→批量；🟡 1.5–3% 或完播低→限 1–2 条试验；🔴 <1.5% 连续 2 条（或单条<1%）→ 砍主题切储备（24h 内上线替换主题）。

## 5. 软著 & 版号版本准备

> 团队具备版号申报落地能力（制作人 19 年全平台工业化经验 + 国风模拟经营赛道深度 + 版号申报实操），相关申报由制作人统筹，不单独设岗。以下为 MVP 阶段需并行预备的版本与资料基线。

### 5.1 软著主体准备
- 软著申请需明确**申请主体**（公司 / 工作室），主体资质与名称需在 MVP 阶段确认，避免后续更名导致的申报返工。
- **待细化**：主体确认、经营范围与软著 / 版号申报资质匹配。

### 5.2 软著版本 & 资料
- 软著需提交**可运行版本 + 源代码（前后各 30 页）+ 设计文档**。
- MVP 阶段产出的"可玩核心循环原型"即可作为软著素材来源；建议在原型首个可运行冻结点同步固化软著版本。
- **待细化**：软著版本冻结点、源代码提取与文档归档清单。

### 5.3 版号版本 & 资料
- 版号申报资料含游戏自审、出版 / 运营单位对接等材料，申报节奏通常晚于 MVP，与长线盯数 / 买量放大阶段协同推进。
- 团队已有版号申报能力可复用，MVP 阶段仅需预留版本与资料接口，不提前投入申报流程。
- **待细化**：版号申报节奏排期、出版单位合作、版本合规自查清单。
